PHP-DI 4

The dependency injection container for humans

PHP-DI helps you using dependency injection in your application, without writing boilerplate code and avoiding the trap of the Service Locator anti-pattern.

Open

  • ZF 1 integration
  • ZF 2 integration

Integrates easily to your project or your framework.

It was also designed to be able to work along with other containers.

PHP-DI Bundle

Practical

Autowiring

80% of the use cases for 0 configuration! 

class Foo
{
    private $bar;

    public function __construct(Bar $bar) {
        return $this->bar = $bar;
    }
}

Annotations

Makes your life so easy in your controllers! 

class Foo
{
    /**
    * @Inject
    * @var Bar
    */
    private $bar;
}

Configuration

For full control! 

$configuration = [

    Bar::class => DI\object(),

    Foo::class => DI\object()
        ->withConstructor(DI\link(Bar::class))

];

Features are "à la carte": mix them all, disable annotation support, do whatever you want.

Modern

PHP-DI reinvented itself for v4.0. Its configuration format has been thought through for the future: PHP 5.4, 5.5 and beyond!

We dropped the YAML format because we do no longer believe it's the most appropriate since:

PHP 5.4

Short arrays: 

return [


];

PHP 5.5

Class name resolution via ::class: 

return [
    Foo::class => DI\object(),

];

Future PHP 5.6

Import functions: 

return [
    Foo::class => object()
        ->withConstructor(link(Bar::class))
];

Furthermore, using PHP instead of YAML comes with other advantages:

  • Auto-completion and refactoring support
  • Define services using callbacks (à la Pimple)
  • You can actually use PHP code like constants, operations, … (instead of writing a custom language that developers have to learn)

Yes, PHP-DI is still compatible with PHP 5.3 (but seriously, upgrade!)

Fast

Lazy-loading

Are you injecting a service that is expensive to build?

Just mark it as lazy and it will be created only when it is used.

Configuration caching

Autowiring (reflection), annotations or PHP configuration have the same impact.

Every definition is cached so that is it not recomputed on each request.
